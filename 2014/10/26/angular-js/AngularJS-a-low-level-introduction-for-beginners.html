title: AngularJS - a low level introduction for beginners date: 2014-11-20 20:08:50 tags: --- AngularJS, commonly referred to as Angular, is an open source framework mainly used for web application that is maintained by Google and the open source community. The goal of Angular is to simplify both development and testing of single page applications (SPA) by providing a framework for client-side model–view–controller (MVC) architecture, along with components commonly used in rich web applications. Angular works by first reading the HTML page, which has embedded into it additional custom tag attributes. Attributes are interpreted as directives telling Angular to bind input or output parts of the page to a model that is represented by standard JavaScript variables. The values of those JavaScript variables can be manually set within the code, or retrieved from static or dynamic JSON resources.

Data-binding

Data-binding in Angular is the simultaneous occurrence of data between the model and view components. The way that Angular implements data-binding lets you utilize the model as the single source of truth in your application. The view is a projection of the model at all times. When the model changes, the view reflects the change, and vice versa.

Expressions

In order to create the views of your application, AngularJS let you execute expressions directly within your pages. In those expressions, you have access Javascript.

Directives

Directives are an extremely powerful feature of the AngularJS framework. They allow you to extend HTML to create a rich web application. Directives let you specify how your page should be structured for the data available in a given scope. AngularJS comes shipped with numerous directives which let you build your basic web application. ngRepeat is a directive which is commonly used in Angular. This directive lets AngularJS create a new set of elements in the dom for each element in a collection. Additionally, Angular comes with a set other built-in directives, like ngBind, ngModel, and ngClass. Much like you create controllers and services, you can create your own directives for Angular to use. When Angular bootstraps your application, the HTML compiler traverses the DOM matching directives against the DOM elements.
